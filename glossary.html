Web Development Glossary

A guide for non-programmers to understand what AI can build

When you ask AI to build interactive tools, it creates them using web technologies. This glossary explains the technical terms so you can better describe what you want and understand what's possible.

When you converse with your coding assistant you should use these terms. Try the interactive examples below!

The Building Blocks: Web Technologies

HTML (HyperText Markup Language)
The language that defines the content and structure of a web page. It specifies what elements appear on the page: headings, paragraphs, buttons, images, etc.
The blueprint of a house that shows where the rooms and doors go.
<h1>Welcome to My Page</h1>
<p>This is a paragraph of text.</p>
<button>Click Me</button>
CSS (Cascading Style Sheets)
The language that defines the visual appearance of a web page. It controls colors, fonts, spacing, sizes, and layout.
The interior design choices: paint colors, furniture styles, and decorations.
h1 {
  color: blue;
  font-size: 24px;
}
button {
  background-color: red;
  padding: 10px;
}
JavaScript
The language that makes web pages interactive. It responds to user actions, updates content dynamically, and creates the "behavior" of the page.
The electrical and mechanical systems that make lights turn on when you flip a switch.
button.onclick = function() {
  alert('You clicked the button!');
}

Page Structure Terms

Element

Any individual component on a web page. A button is an element. A paragraph is an element. An image is an element. Everything you see on a page is made up of elements.

Elements can be styled with many visual properties. You can change an element's font color, background color, border, padding (space inside the element), margin (space outside the element), font size, font family, and much more. These are the kinds of things you can ask AI to adjust when you want to change how something looks.

A div (short for "division") is a generic container element used to group other elements together. Divs are invisible boxes that help organize the page layout.

Elements are like pieces of furniture in a room. A div is like a shelf that holds and organizes other items within it.
<div class="sidebar">
  <h2>Navigation</h2>
  <a href="home.html">Home</a>
  <a href="about.html">About</a>
</div>

Interactions

Hover
What happens when you move your mouse cursor over an element without clicking. Web pages can detect this and respond by changing appearance or showing additional information.
Walking past a motion-sensor light that turns on when you get close.
Try it: Move your mouse over this box
Hover over me! 
.box:hover {
  background-color: red;
  transform: scale(1.05);
}
You can ask for: "When someone hovers over [element], show [information]" or "Change the color when hovering"
Tooltip
A small text label that appears when you hover over an element. Tooltips provide additional information without cluttering the main interface.
The small explanatory placard next to an exhibit in a museum.
Try it: Hover over the element below
Hover for more info This is a tooltip! It provides extra context. 
<span class="tooltip-trigger">Hover for more info</span>
<span class="tooltip-text">
  This is a tooltip! It provides extra context.
</span>
You can ask for: "Show a tooltip with [explanation] when hovering over [element]" or "Add helper text that appears on hover"

User Interface Components

These are the interactive elements users can manipulate. Each example below is fully functional.

Button
A clickable element that triggers an action. Buttons can have different styles to indicate their importance: primary buttons for main actions, secondary for alternatives, and outline buttons for less prominent options.
A doorbell or elevator button that makes something happen when pressed.
Try it: Click any button (these show an alert)
<button>Primary</button>
<button>Secondary</button>
<button>Outline</button>
You can ask for: "Add a button that [does action]" or "Make the submit button more prominent" or "Add a cancel button"
Link (Hyperlink)
Text that navigates to another page or location when clicked. Links are typically underlined and colored differently from regular text. Unlike buttons (which perform actions), links take you somewhere else.
A cross-reference in a book that says "see page 42."
Try it: Click the link

Learn more about web development at MDN Web Docs.

Learn more about web development at <a href="https://developer.mozilla.org">MDN Web Docs</a>
You can ask for: "Add a link to [resource]" or "Make [text] clickable to go to [destination]"
Dropdown (Select Menu)
A menu that expands to show a list of options when clicked. The user can select one option, and the menu collapses to show only their choice.
A filing cabinet drawer that opens to reveal folders inside.
Try it: Click to see options
<select>
  <option>Choose a subject...</option>
  <option>History</option>
  <option>Literature</option>
  <option>Philosophy</option>
  <option>Art History</option>
</select>
You can ask for: "Add a dropdown to select [categories]" or "Let users filter by [criteria] using a dropdown"
Textbox (Text Input / Text Field)
A rectangular area where users can type text. Can be a single line (for short input like names) or multiple lines (for longer content like comments).
A blank line on a paper form where you write your answer.
Try it: Type something
<input type="text" placeholder="Type your name here...">
You can ask for: "Add a textbox where users can enter [information]" or "Let users type their [response/answer/query]"
Checkbox
A small square that can be checked or unchecked. Checkboxes allow users to select multiple options from a list (unlike radio buttons).
A to-do list where you can check off multiple completed tasks.
Try it: Select any that apply (you can choose multiple)
<label>
  <input type="checkbox"> I teach undergraduate courses
</label>
<label>
  <input type="checkbox"> I teach graduate courses
</label>
You can ask for: "Add checkboxes so users can select multiple [options]" or "Let users check off [items] as they complete them"
Radio Buttons
Small circles that can be selected. Unlike checkboxes, radio buttons only allow one choice from a group. Selecting one automatically deselects the others.
Multiple choice questions on an exam where you can only bubble in one answer.
Try it: Select one option (only one allowed)
<label>
  <input type="radio" name="experience"> Beginner - New to this topic
</label>
<label>
  <input type="radio" name="experience"> Intermediate - Some familiarity
</label>
<label>
  <input type="radio" name="experience"> Advanced - Extensive experience
</label>
You can ask for: "Add radio buttons so users must choose one [option]" or "Make it so only one [choice] can be selected at a time"
Modal Dialog (Popup Window)
A window that appears on top of the main page, usually requiring the user's attention before they can continue. The background is often dimmed to focus attention on the modal. Used for important messages, forms, or confirmations.
A person stepping in front of you to ask an important question before you continue walking.
Try it: Click the button to open a modal
<div class="modal">
  <button class="close">&times;</button>
  <h3>This is a Modal Dialog</h3>
  <p>Notice how the background is dimmed...</p>
  <button>Close This Modal</button>
</div>
You can ask for: "Show a popup when users click [button]" or "Display instructions in a modal dialog" or "Ask for confirmation before [action]"
Toggle (Switch)
An on/off control that slides between two states. Toggles are used for binary choices where the change takes effect immediately, without needing a separate "save" button.
A light switch that turns something on or off.
Try it: Click the toggle to switch states
Notifications: Off 
<label class="toggle-switch">
  <input type="checkbox">
  <span class="toggle-slider"></span>
</label>
<span>Notifications: Off</span>
You can ask for: "Add a toggle to enable/disable [feature]" or "Let users turn [setting] on or off with a switch"
Slider (Range Input)
A control that lets users select a value by dragging along a track. Sliders are ideal for selecting from a continuous range of values, like volume, brightness, or a year range.
A volume knob or dimmer switch that you slide to adjust levels.
Try it: Drag the slider
Font size: 50% 
<input type="range" min="0" max="100" value="50">
You can ask for: "Add a slider to adjust [value]" or "Let users select a range between [min] and [max]"
Tabs
A way to organize content into separate sections that users can switch between without leaving the page. Only one tab's content is visible at a time.
Dividers in a binder that let you flip to different sections.
Try it: Click different tabs
This is the Overview tab. It shows introductory content.
This is the Details tab. It shows more in-depth information.
This is the Examples tab. It shows practical demonstrations. 
<div class="tab-buttons">
  <button class="tab-btn">Overview</button>
  <button class="tab-btn">Details</button>
  <button class="tab-btn">Examples</button>
</div>
<div class="tab-content">
  This is the Overview tab...
</div>
You can ask for: "Organize this into tabs" or "Add tabs so users can switch between [sections]"
Accordion
Stacked sections that expand and collapse when clicked. Accordions save space by showing only the headers until users click to reveal content. Common for FAQs and organized lists.
A folded paper fan or a filing system where you pull out one drawer at a time.
Try it: Click any header to expand/collapse
An accordion is a UI pattern where content is hidden by default and revealed when the user clicks on a header. It helps organize information in a compact way.
Accordions work well for FAQs, long lists of information, or any content where users only need to see one section at a time.
Yes! This example allows multiple sections to be open at once, but you can also design it so only one section can be open at a time. 
<div class="accordion">
  <div class="accordion-item">
    <button class="accordion-header">
      What is an accordion?
    </button>
    <div class="accordion-content">
      An accordion is a UI pattern where...
    </div>
  </div>
</div>
You can ask for: "Put this content in an accordion" or "Make the FAQ sections expandable/collapsible"

Putting It All Together

How to Ask for What You Want
When requesting changes or new features, you can now use these terms precisely:
  • "Add a button that submits the form"
  • "Include a link to the course syllabus"
  • "When users hover over a term, show a tooltip with its definition"
  • "Add a dropdown that lets users select their department"
  • "Add a textbox where users can type their response"
  • "Use checkboxes so students can select multiple learning objectives"
  • "Use radio buttons for the quiz so only one answer can be selected"
  • "Show the instructions in a modal dialog when the page loads"
  • "Add a toggle to switch between light and dark mode"
  • "Use a slider to let users adjust the difficulty level"
  • "Organize the content into tabs for different topics"
  • "Put the FAQ in an accordion so users can expand each question"
Remember: Your main job is to describe what you want to see or what you want users to experience, and the AI will handle the implementation. You rarely have to work with the code directly.
